# SP-5 Retro — Daily Tasks + Dashboard Wiring

**Branch:** `main` (work was merged directly, not via a feature branch)
**Merged:** 2026-05-14
**Spec:** `docs/superpowers/specs/2026-05-14-sp5-daily-tasks-design.md`

## What shipped

All §10 acceptance criteria from the spec landed across 9 commits (Phases A–J). Highlights:

- Three tables (`task_weeks`, `daily_tasks`, `user_task_completions`) plus `task_type` enum and `get_task_rollup` RPC, applied live to project `zptxhbblbcaliwezltzp`.
- `lib/tasks.ts` with 22 Vitest assertions covering streak math, completion %, date helpers, and the pre-signup forward-only rule from Q9.
- All four dashboard placeholders wired: `TodaysTasks` (real fetches + optimistic `TaskRow` toggle), `WeeklyStreak` (replaces `WeeklySchedule`, items list removed), `PerformanceOverview` (re-labelled "Daily Tasks", series = completion %), `Tasks Done` and `Active Streak` stat cards.
- Admin `/admin/tasks` per-category scheduler with Live/Draft two-column view, Base UI Dialog drawer for per-day editing (URL-controlled open state), and `/admin/tasks/past` read-only paginated view.
- Toggle endpoint at `POST /api/tasks/[id]/toggle` (insert/delete via single endpoint).
- Playwright smoke for unauth redirects + toggle API shape.

## What went well

- The plan's pre-existing helper patterns (`requireCoach`, `createSupabaseAdminClient`, `as never` write casts) translated directly — no architectural surprises.
- `adjustedRollup` cleanly solves three problems with one transformation: pre-signup days, streak math, and the chart line all use the same input.
- Splitting the dialog primitive plan: dropped the `components/ui/dialog.tsx` wrapper and inlined Base UI directly in `day-drawer.tsx`. YAGNI win.

## What was harder than expected

- The branch story was messy — the spec was committed to `sp5-daily-tasks`, then merged to main, then unrelated work landed on the same branch between sessions, so SP-5 implementation ended up committing directly to main per user direction. Worth a clearer branch protocol next sprint.

## Surprises

- Pre-existing lint errors in `app/(authed)/admin/programs/[id]/edit/page.tsx`, `app/(authed)/settings/page.tsx`, and `components/layout/Sidebar.tsx` (4 problems total, 2 errors). None introduced by SP-5; flagged but not fixed in this sprint to keep scope tight.
- Supabase types regen surfaced `get_task_rollup` automatically — no manual type stub needed.
