SP-4 Data Model

The full schema for exercises, programmes, sessions, progress tracking, and max lifts. Does this look right?

Exercise Library

Global library — not tied to a category. Coach creates exercises once, reuses them across any programme.

exercises
  id · uuid PK
  name · text (e.g. "Back Squat")
  description · text (optional coaching notes)
  video_url · text (optional — SP-6 will expand this)
  muscle_groups · text[] (e.g. ["quads","glutes"])
  tags · text[] (for filtering: "compound", "upper", "lower")
  created_at · timestamptz

Programme Structure (4 tables)

programs
  id · uuid PK
  title · text
  description · text
  category · user_category (A/B/C/D)
  is_active · boolean
  created_at · timestamptz
program_weeks
  id · uuid PK
  program_id · FK → programs
  week_number · int (1, 2, 3…)
  title · text (e.g. "Foundation")
  description · text
  is_free_preview · boolean
program_sessions
  id · uuid PK
  week_id · FK → program_weeks
  session_number · int (1, 2…)
  title · text (e.g. "Lower Body")
  description · text
session_exercises
  id · uuid PK
  session_id · FK → program_sessions
  exercise_id · FK → exercises
  order_index · int
  sets · int
  reps · text ("5" or "8–12")
  pct_of_1rm · int (null = no % prescribed)
  rest_seconds · int
  notes · text

Max Lifts + Progress Tracking

profiles (additions)
  max_lift_squat · numeric (kg)
  max_lift_bench · numeric (kg)
  max_lift_deadlift · numeric (kg)
  max_lift_ohp · numeric (kg)
— user updates these in settings
— used to calculate session weights
user_session_completions
  id · uuid PK
  user_id · FK → profiles
  session_id · FK → program_sessions
  program_id · FK → programs
  week_number · int
  completed_at · timestamptz
— marks a session done
— drives the % completion calc

Plan gating logic

Free: Can view programme overview + any week where is_free_preview = true. Locked weeks show an upgrade prompt.
Basic+: Full programme access. All weeks and sessions unlocked.
Pro: Basic + nutrition (SP-8). No difference in programme access vs Basic.
