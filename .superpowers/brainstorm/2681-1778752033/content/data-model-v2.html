Exercise Library (global, reusable)

From brief §13: "title, muscle_group, tutorial_relation, notes". Sets/reps/weight live on the session prescription, not the exercise itself — exercises are reusable across any session.

exercises
  id · uuid PK
  title · text (e.g. "Back Squat")
  description · text
  muscle_group · text[] (e.g. ["quads","glutes","core"])
  tags · text[] (filtering: "compound", "push", "pull", "lower")
  video_url · text ← placeholder; SP-6 Tutorial Library links here
  created_at · timestamptz

Programme Structure (4 tables)

programs
  id · uuid PK
  title · text
  description · text
  cover_image_url · text ← brief §8
  category · user_category (A/B/C/D) nullable = all
  plan_access · subscription_tier free/basic/pro — brief §8

  created_at · timestamptz
program_weeks

  program_id · FK → programs

  description · text
program_sessions (workouts)
  id · uuid PK
  week_id · FK → program_weeks
  session_number · int
  title · text (e.g. "Lower Body")
  instructions · text ← brief §8
  estimated_duration_mins · int
  completion_rule · text ← brief §8
session_exercises
  id · uuid PK
  session_id · FK → program_sessions

  order_index · int

  weight · text ("70% 1RM" or "60kg") ← brief §13

  rest_seconds · int
  notes · text
  tutorial_id · FK → tutorials nullable — SP-6

  max_lift_bench · numeric (kg) — nullable

User updates these in /settings. Used in session view to calculate actual weight from % of 1RM.

Plan gating — brief §5

Gating is at the program level via programs.plan_access:
Free: Can only see programs where plan_access = 'free'. Others show upgrade CTA.
Basic: Can access programs where plan_access ∈ ('free', 'basic').
Pro: Full access to all programs. + nutrition (SP-8).
